Bypassav之免杀项目改造

#bypassav
Dec 3, 2023

在之前的文章中，发布一篇可以绕过火绒的加载器。当一些特征被记录了应如何改造呢？

本文将隐藏一些可能会被记录为特征，成为溯源判断的辅助信息方法。

pdb 信息

对于 Vs 2022 默认的 Release 模式编译后的文件，会带有 pdb 文件。
pdb 文件将显示编译时的路径，如果在 VS 2022 默认的新建项目编译，会暴露用户名 C:\Users\XXXX\source\xxx.pdb。此处可能会成为溯源的一个重要点。

解决方案：
选中项目名称，按 Alt+Enter，选择 Linker - Debugging 将 Generate Debug Info 改为 No

再次编译，pdb 已经消失。

运行时库

使用 PE-bear 查看，也可以看到这些函数。其中当使用默认的编译选项会带有 api-ms-win-crt-*.dll 这个库，当电脑上没有相关 dll 时，会导致无法运行 exe。

解决方案是编辑属性，选择 配置属性 - C/C++ - 代码生成 - Runtime Library - 选择 Multi-threaded(/MT)

再次编译后，api-ms-win-crt-*.dll 已经消失

添加图标资源

最后可以给程序添加一个图片，以更加真实。
首先，挑选一张合适 icon 图片，使用 Everything 在本地搜索。然后复制到项目目录。

选择资源文件 - 添加 - 资源

点击 导入 选择文件。

再次编译就可以看到图标了。

还有一个很重要的点，这里可以看到 VirtualAlloc、CreateThread、WaitForSingleObject 等字符串显示在其中。
